English Learning: Design Thinking for Beginners

Dialogue

Alice: Bob, I'm starving! We ran out of chips again!

Bob: Oh, Alice, not again! This is a serious problem. We need to 'design think' this!

Alice: Design what, Bob? Just buy more chips!

Bob: No, no! Design thinking is a way to solve problems creatively. First, we need to understand the 'user'.

Alice: The user? Who's the user? Me and you, the snack monsters?

Bob: Exactly! What do we *really* need? When do we run out?

Alice: Hmm. We need snacks like, all the time. And we run out when we forget to check the pantry.

Bob: Good! Now, let's 'ideate' – that means brainstorm ideas!

Alice: Okay, how about a giant snack vending machine in the living room?

Bob: (Laughs) Fun idea! Or, a weekly snack shopping list? Or an alarm on our phones?

Alice: An alarm for snacks? That's genius, Bob!

Bob: See? Then we 'prototype' – try a simple version of the idea! Maybe the alarm reminds us on Friday to check.

Alice: And 'test' it? If we still run out, we try a new idea?

Bob: Exactly! We iterate! Less stress, more chips!

Alice: Wow, Bob. Design thinking for snacks. You're a genius! Now, about that vending machine idea...

Current Situation

Design thinking is a popular approach used by companies and individuals to solve complex problems and create innovative solutions. It's not just for designers anymore! It's a human-centered method that focuses on understanding the people you are designing for (the "users") and then creating solutions that meet their needs. While often used in business, technology, and social innovation, the principles of design thinking are simple enough to apply to everyday problems, like finding creative ways to manage your household or even planning a fun event with friends. It encourages creativity, collaboration, and a willingness to learn from failures.

Key Phrases

  • Design thinking: A creative way to solve problems.
  • Example: We can use design thinking to improve our daily routine.
  • Solve a problem: To find an answer or solution to a difficulty.
  • Example: Let's work together to solve this problem.
  • Understand the user: To know what the people using your product/service truly need or feel.
  • Example: First, we need to understand the user's struggles with this app.
  • Brainstorm ideas (or ideate): To think of many ideas quickly and freely.
  • Example: We should brainstorm ideas for our next team project.
  • Prototype: To make a first or early model of something to test.
  • Example: We can prototype a simple website layout.
  • Test it out: To try something to see if it works or is effective.
  • Example: Let's build a small model and test it out.
  • Creative solution: An original and imaginative answer to a problem.
  • Example: Design thinking helps us find creative solutions.
  • Everyday problems: Common issues or difficulties faced in daily life.
  • Example: You can use these steps for everyday problems too, like organizing your desk.

Grammar Points

  • Present Simple Tense: Used for facts, habits, and general truths.
    • Structure: Subject + Base Verb (or Verb + -s/-es for he/she/it)
    • Examples: "We run out of chips." (a habit) "Design thinking is a way." (a fact) "It helps us." (a general truth)
  • Imperatives: Used to give commands, instructions, or suggestions.
    • Structure: Base Verb (no subject)
    • Examples: "Buy more chips!" "Understand the user." "Test it!"
  • Modal Verbs (can, should):
    • Can: Expresses ability or possibility.
    • Example: "We can prototype a simple solution."
    • Should: Expresses recommendation or advice.
    • Example: "We should design think this problem."
  • "Let's" (Let us): Used to make a suggestion that includes the speaker and listener.
    • Structure: Let's + Base Verb
    • Examples: "Let's brainstorm ideas." "Let's test it out."
  • Wh-Questions: Words like "Who," "What," "When," "Where," "Why," "How" used to ask for specific information.
    • Examples: "Who's the user?" "What do we really need?" "When do we run out?" "What do you need?"

Practice Exercises

Exercise 1: Fill in the blanks with a suitable key phrase from the list above.

  1. If you want to find a new way to do something, you need to _________.
  2. Before you make something big, you can _________ a small version.
  3. We need to _________ to know what people really want.
  4. After we have many ideas, we should _________ to see if they work.
  5. Design thinking helps us find _________.

Answers:

  1. design think
  2. prototype
  3. understand the user
  4. test it out
  5. creative solutions

Exercise 2: Match the sentence parts.

  1. Let's design think _________
  2. First, understand _________
  3. We need to brainstorm _________
  4. This method helps us find _________
  5. Let's prototype a _________

Answers:

  1. 1. C. our snack problem.
  2. 2. B. the user's needs.
  3. 3. D. ideas.
  4. 4. A. creative solutions.
  5. 5. E. small version first.

Exercise 3: Make a complete sentence using the given words. Remember to use the correct grammar.

  1. (we / should / design think / this problem)
  2. (always / I / forget / my keys)
  3. (let's / brainstorm / new ideas)
  4. (it / helps / us / solve problems)
  5. (what / you / need / do?)

Answers:

  1. We should design think this problem.
  2. I always forget my keys.
  3. Let's brainstorm new ideas.
  4. It helps us solve problems.
  5. What do you need?
